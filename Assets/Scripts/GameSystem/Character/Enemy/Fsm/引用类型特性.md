## 实现原理

这个方法利用了 C# 中**引用类型**的特性。在 C# 中，`List<T>` 是一个引用类型，当你将一个 List 作为参数传递给方法时，传递的是这个 List 对象的**引用**（地址），而不是 List 的副本。

## 具体实现代码

```csharp
public void GetAllStates(List<FsmState<T>> results)
{
    if (results == null)
    {
        throw new GameFrameworkException("Results is invalid.");
    }

    results.Clear();  // 先清空传入的列表
    foreach (KeyValuePair<Type, FsmState<T>> state in m_States)
    {
        results.Add(state.Value);  // 将每个状态添加到传入的列表中
    }
}
```

## 工作流程

1. **调用前**：调用者创建一个 List 对象
   ```csharp
   List<FsmState<T>> myStates = new List<FsmState<T>>();
   ```

2. **调用时**：将 List 的引用传递给方法
   ```csharp
   fsm.GetAllStates(myStates);
   ```

3. **方法内部**：
    - 先清空传入的 List（`results.Clear()`）
    - 遍历内部存储状态的字典 `m_States`
    - 将每个状态添加到传入的 List 中（`results.Add(state.Value)`）

4. **调用后**：`myStates` 中就包含了所有的状态

## 为什么这样设计

- **避免内存分配**：不需要每次调用都创建新的 List
- **引用传递**：直接修改传入的 List 对象，调用者可以看到修改后的结果
- **性能优化**：减少 GC 压力，这在游戏开发中非常重要

这种设计模式在 Unity 和游戏开发中非常常见，比如 `Transform.GetComponentsInChildren<T>(List<T> results)` 也是同样的原理。
